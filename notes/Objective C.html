4.2 @interface

Describes the class, its data components and methods 

@interface NewClassName: ParentClassName
{
  int member;
  int another_member;
}

-(void) method1;
+(int) method2;
@end

4.3 @implementation

Contain code that implements the class and instance methods.

4.4 - (void) methodName: (int)n;

- is the method type. - is for instance methods, + indicates a class method. (void) is the return type. If you don't specify a return type, id is the default.

4.5 synthesized accessor methods

Automatic get and set. Add @property to your class interface and @synthetize in your implementation. 

@interface MyClass : NSObject
{
  int member;
  int another_member;
}

@property int member, another_member;

@end

@implementation

@synthesize member, another_member;

@end

4.6 dot

As of Objective-C 2.0 you can access properties using the dot operator. 

[myInstance property]
myInstance.property

4.7 @class

You can either #import an header file or use @class MyClass. @class is more efficient because the compiler does not need to process the entire header file.

6 id

id is a data type used to store an object of any type. In a sense, it is a generic object type.

We don't declare all objects as id because static typing allows the compiler to check for errors. And it is better to get errors during the compilation time than during execution time.

14 protocols

A protocol is a list of methods that is shared among classes. 

@interface MyClass: NSObject <NSCopying, NSCoding>

And you need to implement the methods to conform to the protocol

Here Drawing protocol requires you to implement paint in your classes. outline can be implemented too but it is optional. 

@protocol Drawing
-(void) paint;
@optional
-(void) outline;
@end

You help the compiler to check for compliance of id variables and protocols. 

id <Drawing> currentObject;

An informal protocol is really a category that lists a group of methods but does not implement them. They can also be referred as abstract protocols.

15 compound literal

A compound literal is a type name enclosed in parentheses followed by an initialization list. 

(struct date) { .month = 7, .day =2, .year = 2004 };
int intPtr *;
intPrt = (int [100]) {[0] = 1, [50] = 50, [99] = 99 };
